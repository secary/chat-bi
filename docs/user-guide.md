## 2. 产品是什么、能做什么

**零眸智能 ChatBI** 是一个面向银行业务场景的**对话式数据分析 Demo**。用户用**中文自然语言**提问，系统在后台通过 **Agent + 技能（Skill）** 调用确定性脚本访问 **MySQL** 中的演示业务数据与语义层元数据，再通过网页以**文字、思考步骤、图表、KPI 卡片**等形式返回结果。

典型能力包括：

- **问数**：按指标、维度、时间范围查询销售额、订单、客户等（受语义层约束的只读查询）。
- **环比 / 对比**：按区域、渠道等维度做月度对比分析（由对应 Skill 支持）。
- **经营决策建议**：在计算事实指标后，按规则生成可读的建议文本（ often 以 Markdown 形式展示）。
- **语义别名维护**：为指标或维度登记「别名 → 标准名」映射，便于后续自然语言命中。
- **文件与图像**：上传表格类文件做结构校验与预览；上传图片时可配合视觉模型抽取表格信息纳入上下文（取决于后端配置）。
- **多专线协作**：可选开启多 Agent 专线顺序执行并汇总（管理员可配置专线与技能组合）。
- **会话与记忆**：多轮对话按用户隔离；系统可注入长期偏好与近期摘要类提示（可关闭）。
- **数据仪表盘**：独立页面展示当前业务库的聚合 KPI 与固定图表视图。
- **PDF 报告**：将当前会话导出为服务端生成的 PDF（含摘要与图表嵌入，具体取决于运行环境）。

本项目定位为 **MVP / Demo**，非完整企业级 BI 平台：权限模型为演示级，详细约束见 `[README.md](../README.md)` 与各环境变量说明。

## 3. 技术栈与运行方式（概要）


| 层级   | 说明                                            |
| ---- | --------------------------------------------- |
| 前端   | React + TypeScript + Vite，图表使用 ECharts        |
| 后端   | FastAPI，对话接口使用 **SSE（服务端推送）** 流式返回思考与结果       |
| 模型调用 | LiteLLM 统一对接配置的 LLM                           |
| 数据   | MySQL 8.0，演示库、应用库、管理配置等分库分表（初始化见 `database/`） |


**启动方式**请优先参考仓库根目录 `[README.md](../README.md)`：

- **Docker 全栈（生产式本地）**：一键 MySQL + 后端 + 前端静态资源，默认端口见 README 中的表格。
- **Docker 开发热更新**：前后端源码挂载，端口与生产式本地不同，可并行。
- **宿主机开发**：本地运行后端与前端，MySQL 仍建议用 Docker。

环境变量模板见 `.env.example`；本地开发常配合 `.env.dev` / `env.dev`（项目约定见 `[AGENTS.md](../AGENTS.md)`）。

## 4. 登录、角色与鉴权开关

- **默认管理员账号**（数据库按 `database/init.sql` 首次初始化后）：用户名 `admin`，密码 `admin123`。对外演示或生产部署前**务必修改密码**，并通过「用户管理」创建业务账号。
- **角色**：至少区分普通用户与 **admin**。左侧导航中「多 Agents 管理、技能管理、数据源、LLM、用户管理」仅 **admin** 可见。
- **开发环境可关闭登录**：当后端 `CHATBI_AUTH_ENABLED` 与前端 `VITE_AUTH_ENABLED` 配置为关闭时，界面可能显示「开发环境：用户登录已关闭」，此时仍以种子用户身份访问接口（细节以部署配置为准）。

## 5. 界面总览：路由与侧栏

登录后（若开启鉴权），主布局为**左侧导航 + 右侧内容区**。


| 路由路径            | 菜单名称        | 谁能看见   | 主要内容                        |
| --------------- | ----------- | ------ | --------------------------- |
| `/`             | 对话          | 所有登录用户 | 自然语言问答、会话列表、上传、多专线开关、PDF 导出 |
| `/dashboard`    | 仪表盘         | 所有登录用户 | 业务库聚合 KPI、固定图表、语义层资产规模      |
| `/multi-agents` | 多 Agents 管理 | admin  | 多专线编排、每轮上限、专线标签与技能多选        |
| `/skills`       | 技能管理        | admin  | Skill 启用/禁用等（与后端扫描注册表一致）    |
| `/data-sources` | 数据源管理       | admin  | 业务库连接配置、测试连接、当前生效连接展示       |
| `/llm`          | LLM 配置      | admin  | 模型与 API 等运行时参数、当前生效配置摘要     |
| `/users`        | 用户管理        | admin  | 用户账号的创建与维护                  |
| `/login`        | （登录页）       | 未登录    | 用户名密码登录                     |


未匹配的路径会重定向到首页对话 `/`。

## 6. 各页面功能说明

### 6.1 对话页（`/`）

这是核心页面，分为三个区域：**会话侧栏**、**顶部工具栏**、**消息与输入区**。

**会话侧栏**

- **会话列表**：展示历史会话标题（标题会随最近一次用户提问更新，便于辨认）。
- **新对话**：创建新的会话线程，与旧会话上下文隔离。
- **删除会话**：删除当前列表中的某条会话；若删的是当前选中会话，会切换到其他会话或新建会话。
- **收起 / 展开**：缩窄侧栏以腾出阅读空间，偏好可保存在浏览器本地。

**顶部工具栏**

- **多专线协作**：开关打开后，本轮对话可走「多 Agent 专线」编排（具体专线与技能由管理员在「多 Agents 管理」中配置）。开关状态通常保存在本地，刷新后保持。
- **导出 PDF 报告**：基于当前会话生成 PDF 并下载。生成依赖后端环境（如字体与 PDF 引擎）；失败时界面仅能通过日志或网络面板排查。
- **数据源连接 ID（可选）**：填入数字 ID 时，本轮请求可指定使用哪一条已配置的数据源连接；留空则使用系统默认连接（与「数据源管理」中的默认策略一致）。

**消息区**

- **用户消息 / 助手消息**：助手侧可能包含：
  - **思考步骤**：可折叠的分析过程，便于理解「做了什么」。
  - **文本结论**：支持常见 Markdown 层级（标题、列表等）的阅读样式。
  - **图表**：柱状、折线、饼图等，一般支持图例、提示框、缩放或时间轴筛选（依图表类型而定）。
  - **KPI 卡片**：关键指标以卡片形式展示，可能带成功 / 警告 / 危险等语义色。
  - **错误提示**：脚本异常、超时或配置问题会以错误类型消息呈现。
- **助手正在生成**：若离开页面再返回、或 SSE 暂时断开，界面可能显示「处理中」类提示，并尝试拉取已落库的助手消息，避免空白误解。

**输入区**

- **文本输入**：输入业务问题后发送。支持常见「短问法」，例如「1–4 月销售额排行」。
- **记忆提示 chip**：若后端返回「建议继续追问」类短语，会以按钮形式出现在输入框上方，点击即发送该句。
- **附件上传**：
  - 支持点击选择或拖拽。
  - **表格类**（如 CSV、Excel）：上传成功后，输入框会预填一条引用服务端路径的说明文案，用户确认后可发送；后续 Skill 可对文件做校验或预览（是否写库由产品设计决定，Demo 可能仅校验）。
  - **图像类**（PNG / JPG / WebP）：预填文案会请求将图像纳入分析；是否做表格抽取取决于后端视觉链路是否启用。

**使用建议**

- 先从不带附件的明确指标问题开始，确认数据源与 LLM 已配置正确。
- 需要指定数据库连接时，先在「数据源管理」配置并记下 ID，再在顶部填写。
- 开启「多专线协作」后延迟可能增加，适合复杂分析；简单问数可关闭以缩短路径。

### 6.2 仪表盘页（`/dashboard`）

**用途**：不经由对话，快速浏览当前**生效业务库**的聚合概况。

**典型内容包括**（以实际接口返回为准）：

- **KPI 卡片**：如销售总额、订单明细条数、数据时间范围、覆盖区域数等。
- **图表**：如区域销售占比（饼图）、按月销售柱状图、客户活跃度相关柱状图等。
- **语义层资产**：语义相关表的规模统计列表。
- **警告区域**：若库表缺失或数据异常，可能在顶部以警告列表提示。

进入页面会自动请求后端概览接口；若出错会显示错误文案，需检查登录是否有效、后端与数据库是否可用。

---

### 6.3 多 Agents 管理页（`/multi-agents`，仅 admin）

**用途**：配置「多专线协作」使用的 **registry**：每条专线包含标签、角色提示、关联的多个 Skill 等，并可设置每轮调用的专线上限。

**操作要点**

- 修改后保存，后端会**原子写入**仓库中的注册表文件；保存内容需通过服务端校验（例如专线 id、至少一条专线、技能 slug 必须存在于当前 Skill 扫描结果）。
- 与普通「技能管理」配合：这里选中的 Skill 必须是系统已注册的 Skill。

---

### 6.4 技能管理页（`/skills`，仅 admin）

**用途**：查看与管理 Skill 的启用状态等（与后端 `scan_skills` / 注册表一致）。禁用某 Skill 后，Agent 不应再执行该 Skill（具体以后端实现为准）。

---

### 6.5 数据源管理页（`/data-sources`，仅 admin）

**用途**：维护连到 **MySQL 业务数据** 的连接串与参数，支持测试连接。

**说明**

- 页面顶部通常会展示**当前生效**、用于 Skill 脚本执行的连接摘要（默认连接优先，具体规则以后端为准）。
- 对话页填写的「数据源连接 ID」即此处配置的某条记录的 ID。

---

### 6.6 LLM 配置页（`/llm`，仅 admin）

**用途**：配置 LiteLLM 使用的模型、API Base、Key 等（具体字段以后端管理接口为准）。

**说明**

- 页面会展示**当前生效**的模型与连接摘要，便于确认运行时真实使用的参数来源（环境变量与数据库合并策略由后端处理）。

---

### 6.7 用户管理页（`/users`，仅 admin）

**用途**：创建与管理登录账号、角色等（Demo 级）。用于给团队成员开账号，或替换默认 `admin` 的高风险密码。

## 7. 典型使用场景（怎么问、期望看到什么）

下列问题仅为示例，实际用语可根据语义层与 Skill 覆盖范围调整。


| 场景      | 你可以怎么说（示例）                         | 界面可能出现的内容                     |
| ------- | ---------------------------------- | ----------------------------- |
| 排行 / 趋势 | 「2026 年 1–4 月各区域销售额排行」             | 表格或结构化结果 + 柱状图 / 折线图 + KPI    |
| 环比对比    | 「各区域销售额环比」「4 月和 3 月毛利率对比」          | 对比表格、分组柱状图、环比 KPI 卡片          |
| 决策建议    | 「基于当前数据给出经营决策建议」或结合指标的问题           | 以 Markdown 为主的建议正文，可能伴随 KPI   |
| 别名维护    | 「把『营收』登记为销售额这条指标的别名」               | 助手确认写入语义别名表的流程类回复（需 Skill 支持） |
| 复合意图    | 「查询本月华东销售额并给出建议」                   | 可能出现多段思考或多步 Skill 的结果串联       |
| 上传表格    | 拖拽 `data/chatbi_sales.csv` 后发送预填提示 | 校验结果或预览型 JSON，取决于 Skill       |
| 上传图像    | 上传表格截图并要求「读取图中表格」                  | 若视觉链路开启，助手结合抽取内容作答            |


若回答含糊或不符合预期，优先检查：**LLM 是否可用**、**数据源是否为演示库**、**是否选对了数据源连接 ID**、**相关 Skill 是否启用**。

## 8. 限制与已知约束（用户视角）

- **演示数据**：默认仅为 MVP 演示，指标与维度以语义层与初始化 SQL 为准。
- **安全边界**：问数类 Skill 设计为只读 `SELECT`；别名写入仅限受控表。仍不应把真实生产敏感数据直接接入未经评估的环境。
- **PDF 与中文**：PDF 导出在缺少中文字体的环境下可能出现乱码，推荐在 Docker / Linux 镜像或已安装字体的环境中使用。
- **网络与超时**：LLM 或数据库不稳定时，可能出现错误消息或长时间等待。
- **文档与代码漂移**：功能迭代较快，若界面与本文不一致，以当前仓库 `[README.md](../README.md)` 与 `[docs/plans/current-sprint.md](plans/current-sprint.md)` 为准。
